# Implementation Plan: Vehicle Selector & Sort Order

**Branch**: `010-vehicle-selector-sort` | **Date**: 2026-04-21 | **Spec**: [spec.md](spec.md)

## Summary

Replace the current per-tab toolbar vehicle picker with a rich `VehiclePickerCard` component at the top of each main tab. Introduce a shared `VehicleSelectionStore` (`@Observable`) injected at app root so vehicle selection is synchronized across all tabs. Add a `VehicleSortOrder` preference (Alphabetical / Date Added / Last Used / Custom drag order) to Settings, persisted via `UserDefaults`.

## Technical Context

**Language/Version**: Swift 5.9+
**Primary Dependencies**: SwiftUI, SwiftData, Observation framework (all Apple-provided)
**Storage**: SwiftData (existing Vehicle/FillUp models — no schema changes); UserDefaults for sort preference and selected vehicle ID
**Testing**: Manual simulator testing (XCTest not in scope for this feature)
**Target Platform**: iOS 17.0+

**Performance Goals**: Card renders instantly; sort/selection changes reflect within 1 frame
**Constraints**: Offline-only; no new third-party dependencies; no SwiftData schema changes
**Scale/Scope**: Affects 3 main tab views + SettingsView; introduces 3 new files, modifies 5 existing files

## Implementation Notes

### VehicleSortOrder.swift

```swift
enum VehicleSortOrder: String, CaseIterable, Identifiable {
    case alphabetical = "alphabetical"
    case dateAdded    = "dateAdded"
    case lastUsed     = "lastUsed"
    case custom       = "custom"

    var id: String { rawValue }
    var label: String { /* user-facing name */ }
}
```

### VehicleSelectionStore.swift

`@Observable final class VehicleSelectionStore`

- `var selectedVehicle: Vehicle?` — shared selection
- `var sortOrder: VehicleSortOrder` — backed by `UserDefaults["vehicleSortOrder"]`
- `var customOrder: [UUID]` — backed by `UserDefaults["customVehicleOrder"]` (JSON-encoded)
- `func sortedVehicles(_ vehicles: [Vehicle]) -> [Vehicle]` — returns sorted list per current preference
- `func selectVehicle(_ vehicle: Vehicle)` — sets `selectedVehicle`, updates `lastUsedAt`, persists ID
- `func restoreSelection(from vehicles: [Vehicle])` — called on launch; matches stored UUID
- `func updateCustomOrder(_ vehicles: [Vehicle])` — saves new drag order

### VehiclePickerCard.swift

A `View` struct taking `vehicle: Vehicle`, `isInteractive: Bool`, `onTap: () -> Void`.

- Circular photo (64×64): `vehicle.photoData` → `Image` with `.clipShape(Circle())`; fallback: `Image(systemName: "car.fill")` on accent circle
- Bold vehicle name
- Odometer reading: computed as `max(fillUps.map(\.odometerReading))` or `initialOdometer`
- Chevron: only shown when `isInteractive == true`
- Card background: `Color(.secondarySystemGroupedBackground)` with corner radius

### Tab Integration Pattern

Each of FillUpListView, CostListView, SummaryTabView:

1. Access store: `@Environment(VehicleSelectionStore.self) private var store`
2. Remove `@State private var selectedVehicle: Vehicle?` (use `store.selectedVehicle`)
3. Remove toolbar `.principal` picker
4. Add `VehiclePickerCard` as the first `Section` (no title) inside the `List`
5. Tap action on card → set `showVehiclePicker = true` → sheet with sorted vehicle list

### SettingsView Vehicle Order Section

After the existing Categories section, add:

```
Section("Vehicle Order") {
    Picker("Sort By", selection: $store.sortOrder) {
        ForEach(VehicleSortOrder.allCases) { order in
            Text(order.label).tag(order)
        }
    }
    if store.sortOrder == .custom {
        NavigationLink("Edit Order") {
            VehicleReorderView()
        }
    }
}
```

`VehicleReorderView` is a `private struct` at bottom of SettingsView.swift — a `List` of vehicles with `.onMove` that calls `store.updateCustomOrder(_:)`.
