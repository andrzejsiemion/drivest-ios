# Implementation Plan: Odometer Chart on Statistics Page

**Branch**: `024-odometer-chart` | **Date**: 2026-04-27 | **Spec**: [spec.md](spec.md)
**Input**: Feature specification from `specs/024-odometer-chart/spec.md`

## Summary

Add a line chart to the Statistics tab that plots odometer readings (Y-axis) against time (X-axis) for the currently selected vehicle. The user controls the visible time range via a segmented picker (Last Month / Last Year / All Time); the Y-axis auto-scales to fit the visible data. Built entirely with Apple's Swift Charts framework ‚Äî no new dependencies. Data sourced from existing `FillUp` records.

## Technical Context

**Language/Version**: Swift 5.9+
**Primary Dependencies**: SwiftUI, Swift Charts (Apple, iOS 16+ ‚Äî fully stable on iOS 17)
**Storage**: SwiftData (existing schema ‚Äî no migrations required)
**Testing**: XCTest (unit tests for `OdometerDataPoint` computation and filtering)
**Target Platform**: iOS 17.0+ (iPhone & iPad, portrait and landscape)
**Project Type**: iOS mobile app (SwiftUI / MVVM)
**Performance Goals**: Chart renders within 300 ms on a device with up to 500 fill-up records (SC-004)
**Constraints**: No new third-party packages; no server calls; offline-capable
**Scale/Scope**: Single chart view within existing Statistics tab; 2 new files, 2 modified files
