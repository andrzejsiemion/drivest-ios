# Implementation Plan: Toyota Odometer Integration

**Branch**: `021-toyota-odometer` | **Date**: 2026-04-24 | **Spec**: specs/021-toyota-odometer/spec.md

## Summary

Add a Toyota Connected Services integration that fetches the current odometer reading from Toyota's EU API (`ctpa-oneapi`) and pre-fills the odometer field when logging a fill-up. Users authenticate with their MyToyota account (email + password). The app handles the full OAuth2 login flow internally and stores only the refresh token in Keychain. Architecture mirrors the existing Volvo integration exactly.

## Implementation Phases

### Phase 1 — Core Service Layer

**Goal**: Working API client that can login and fetch odometer; no UI yet.

**1. `ToyotaAPIConstants.swift`**
- Hardcode: `clientID`, `basicAuthHeader`, `apiKey`, `appVersion`, `brand`, `tokenURL`, `telemetryURL`
- Keychain accessors: `isConfigured`, `savedUsername`

**2. `ToyotaAPIClient.swift`**
- `login(username:password:)` — POST to token endpoint with `grant_type=password`
- `refreshAccessToken(refreshToken:)` — POST with `grant_type=refresh_token`
- `fetchOdometer(vin:accessToken:)` — GET `/v3/telemetry` with required headers
- Per-request headers: `x-guid` and `x-correlationid` as fresh UUIDs, `x-appversion`, `x-brand`
- `ToyotaAPIError` enum: `loginFailed`, `tokenRefreshFailed`, `requestFailed`, `unexpectedResponse`
- Each error has `errorDescription` (technical) and `userMessage` (UI-friendly)

**3. `ToyotaOdometerService.swift`**
- `@Observable` class with `isFetching: Bool` and `fetchError: String?`
- `fetchOdometer(vin:)` — orchestrates refresh + fetch; returns `(km: Int, syncedAt: Date)?`

### Phase 2 — Data Model

**4. `Vehicle.swift`**
- Add `var toyotaLastSyncAt: Date?`
- SwiftData handles migration automatically (optional property)

### Phase 3 — Settings UI

**5. `ToyotaSettingsView.swift`**
- **Connected state**: status indicator, username display, "Disconnect" button
- **Disconnected state**: `TextField` for email, `SecureField` for password, "Sign In" button with spinner
- Disclaimer: *"Toyota Connected Services (EU only). Unofficial API — may stop working without notice."*
- On sign-in: call `ToyotaAPIClient.login()`, save `toyota.refreshToken` + `toyota.username` to Keychain
- On disconnect: delete both Keychain keys

**6. `IntegrationsView.swift`**
- Add `NavigationLink` to `ToyotaSettingsView`
- Show green status dot when `ToyotaAPIConstants.isConfigured`

### Phase 4 — Vehicle Detail

**7. `VehicleDetailView.swift`**
- Add Toyota section (conditional on `ToyotaAPIConstants.isConfigured && vehicle.vin != nil`)
- Show relative "Last sync" time or "Not yet synced"

### Phase 5 — Fill-Up Integration

**8. `AddFillUpViewModel.swift` / `EditFillUpViewModel.swift`**
- Add `let toyotaService = ToyotaOdometerService()`
- Add `fetchToyotaOdometer()`: guard vin + isConfigured → fetch → convert units → write to odometerText → update toyotaLastSyncAt

**9. `AddFillUpView.swift` / `EditFillUpView.swift`**
- Add Toyota fetch button on odometer row (same pattern as Volvo button)
- Show spinner from `toyotaService.isFetching`
- Show red error from `toyotaService.fetchError`
- If both Volvo and Toyota are configured: show both buttons

## Key API Details (from research.md)

| Parameter | Value |
|---|---|
| Token URL | `https://b2c-login.toyota-europe.com/oauth2/realms/root/realms/tme/access_token` |
| API base URL | `https://ctpa-oneapi.tceu-ctp-prd.toyotaconnectedeurope.io` |
| Odometer endpoint | `GET /v3/telemetry` |
| VIN delivery | Header: `vin: {vin}` |
| Client ID | `oneapp` |
| Basic auth header | `basic b25lYXBwOm9uZWFwcA==` |
| API key (hardcoded) | `x-api-key: [TOYOTA_API_KEY]` |
| Odometer JSON path | `payload.odometer.value` (Int, km) |
